## Upstream license terms (checked 2026-06-21)

- **HaGeZi** and **OISD** — GPL-3.0. Neither imposes a permission requirement
  beyond GPL; Lava references them source-url-only and does not redistribute the
  list bytes.
- **AdGuard DNS Filter** — GPL-3.0; the same source-url-only posture as
  HaGeZi/OISD. ("AdGuard" is a commercial trademark, used nominatively to
  identify the source only, with no implied endorsement.)
- **1Hosts** — MPL-2.0 (weak / file-level copyleft); under source-url-only the
  share-alike obligation is not triggered.
- **The Block List Project** — Unlicense (public-domain dedication); nothing
  required.
- **Phishing.Database** — MIT; Lava records the license and links to the upstream
  source URL, which carries the MIT copyright/permission notice, and does not
  mirror or modify the bytes.
- **StevenBlack** — MIT, but the hosts distributions *aggregate* multiple
  upstreams each under its own license; Lava links only to StevenBlack's
  published source URLs and never mirrors or modifies the aggregated bytes.

## Copyleft sources and catalog scope

The copyleft sources above (GPL-3.0, MPL-2.0) are source-url-only and off by
default. Under source-url-only Lava never conveys the bytes, so neither the GPL
nor the MPL share-alike obligation is triggered. Such sources are off by default
because each ships `defaultEnabled: false` — the fresh-install set
(`DefaultCatalog.recommendedDefaultSourceIDs`) is Block List Basic only — so
default-enabling one is a deliberate code change, not a config flip. Whether a
source is in the catalog at all is a product decision; the canonical catalog
manifest carries a per-source `counsel_status` review annotation for bookkeeping
(not a runtime gate). Aggregated / meta-lists (e.g., StevenBlack, OISD) compile
multiple upstreams under their own license; Lava links only to their published
source URLs and never mirrors or modifies the bytes.
